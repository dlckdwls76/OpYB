프로젝트 개요 및 종료 조건

  • 프로젝트 개요: 탑다운 뷰 기반의 배틀로얄 멀티플레이어 게임
  • 게임 종료 조건:
    • 한 사람만 살아남고, 모든 캐릭터가 부활 중(사망 상태)일 때 게임 종료
    • 솔로 모드 기반 최후의 1인 생존 방식

프로토타입 구현 내용 (캐릭터/전투)

  • 캐릭터 생성 추후 로비에서 원하는 캐릭터를 선택하고 플레이를 누르면 해당 캐릭터가 생성(SpawnActor)되어 조작할 수 있도록 기본 로직을 구현해 두었습니다.
  • 기본 사격 마우스 좌클릭 시 Projectile(발사체) 액터를 스폰(Spawn)하고 날려 보내는 방식으로 기본 총알 발사를 구현했습니다.
  • 궁극기 시스템 기본 공격 적중 시 내부 카운트 변수를 증가시켜 3회 적중 시 게이지가 100% 차도록 로직을 작성하였고, 조건문을 통해 궁극기를 사용할 수 있는 상태로 전환되게 구현했습니다.

프로토타입 구현 내용 (시스템/멀티)

  • 캐릭터 사망 및 인원 표시 체력이 0이 되어 사망 시 GameState를 통해 전체 남은 인원 수 변수를 감소시키고 이를 모든 플레이어의 UI에 갱신하도록 구현했습니다.
  • 사망 타이머 사망 시 남은 부활 시간을 표시하기 위해 Timer Manager(타이머 핸들러)를 사용하여 UI에 실시간으로 남은 시간이 카운트다운 되도록 구현했습니다.
  • UI 동기화 (체력/탄창 vs 궁극기) 체력 바와 탄창 바는 Replication(리플리케이션)을 사용해 네트워크상에서 모든 클라이언트에 동기화시켰고, 궁극기 게이지는 상대가 알지 못하는 전략적 요소로 쓰기 위해 동기화하지 않고 로컬에서만 갱신하도록 구현했습니다.
  • 배틀로얄 자기장 맵 중앙을 기준으로 스케일이 서서히 줄어드는 액터를 배치하고, 영역 밖 플레이어에게 Overlap 이벤트와 타이머를 통해 지속 데미지(DoT)를 주도록 구현했습니다.
  • 카메라 장애물 투명화 카메라와 캐릭터 사이에 위치한 오브젝트를 LineTrace(라인 트레이스)로 감지한 뒤, 머티리얼의 투명도(Opacity) 파라미터를 조절하여 플레이어의 시야를 가리지 않도록 구현했습니다.

핵심 게임 플레이

  • 게임에 접속하면 로비에서 캐릭터 선택
  • 매칭 씬으로 전환
  • 매칭이 모두 잡히면 전투 진행
  • 시작 시 남은 인원이 좌측 상단에 표시됨
  • 빠른 템포로 자기장이 줄어듦
