# 언리얼 엔진 탑다운 슈팅(총쏘기) 구현 정리

본 문서는 멀티플레이어 환경(리슨 서버)을 고려한 탑다운 시점의 슈팅 시스템 구현 과정과 트러블슈팅 내역을 노션(Notion)에 기록하기 쉽도록 정리한 가이드입니다.

---

## 1. 핵심 함수 이름 및 내용

### 🎮 Player Controller (`OpYBPlayerController.cpp`)
* **`SetupInputComponent()`**
  * **내용:** 언리얼의 향상된 입력(Enhanced Input) 시스템을 사용하여, 마우스 클릭 액션(`ShootAction`)을 C++ 함수에 연결(Bind)합니다.
  * **핵심:** 트리거 이벤트(`ETriggerEvent`)를 통해 발사 타이밍을 조절합니다.
    * `Started`: 클릭하는 순간 단발 발사
    * `Triggered`: 누르고 있는 동안 연속 발사
    * `Completed`: 클릭했다가 손가락을 뗄 때 발사

* **`Shoot()`**
  * **내용:** 마우스 클릭이 감지되었을 때 가장 먼저 호출됩니다. 자기가 조종 중인 캐릭터(`OpYBCharacter`)를 찾아 `AttemptShoot()`을 실행하라고 명령을 내립니다.

### 🏃 Character (`OpYBCharacter.cpp`)
* **`AttemptShoot()`**
  * **내용:** 클라이언트(플레이어) 측에서 발사를 시도하는 함수입니다.
  * **핵심:** **쿨타임(Fire Rate) 검사**를 수행합니다. 현재 시간과 마지막 발사 시간을 비교하여(`CurrentTime - LastFireTime >= FireRate`), 발사 조건이 만족되었을 때만 총알의 스폰 위치(가슴팍)를 계산하고 서버 측 함수(`ServerShoot`)를 호출합니다.

* **`ServerShoot_Implementation()`**
  * **내용:** 클라이언트의 요청을 받아 **서버에서만** 실제로 총알 엑터(`AOpYBProjectile`)를 월드에 생성(Spawn)하는 함수입니다. (멀티플레이어 동기화 필수)
  * **핵심:** `SpawnActorDeferred` (지연 생성) 함수를 사용하여 총알이 물리 세상에 완전히 나타나기 전에 필요한 초기 설정(누가 쐈는지 등)을 미리 주입합니다.

### 🎯 Projectile (`OpYBProjectile.cpp`)
* **`OnOverlapBegin()` / `OnHit()`**
  * **내용:** 총알이 다른 오브젝트와 충돌했을 때 불리는 함수입니다.
  * **핵심:** 총알이 **자신을 쏜 사람의 몸(Pawn)을 통과하도록 Overlap 처리**를 하고, 벽이나 적에게 닿았을 때는 데미지를 입히고 `Destroy()`로 폭발(파괴)시킵니다. 멀티플레이어 환경에서는 파괴 로직 역시 `HasAuthority()`를 통해 **서버에서만** 처리되도록 해야 합니다.

## 2. 발생했던 문제들(실수)과 해결 방법

### 🚨 문제 1: 총 쏠 때마다 무조건 '더블 클릭'을 해야만 총알이 나가는 현상
* **원인:** 언리얼 엔진의 뷰포트 포커스(마우스 캡처) 문제였습니다. 마우스 커서를 화면에 띄워두는 탑다운 게임에서, 입력 모드를 `GameOnly`로 해두면 에디터가 화면 활성화를 위해 첫 번째 클릭을 꿀꺽 삼켜버립니다.
* **해결 내용:** `OpYBPlayerController`의 `BeginPlay()`에서 입력 모드를 **`GameAndUI`**로 변경하고, 캡처 중 커서가 숨겨지는 옵션(`SetHideCursorDuringCapture(false)`)을 꺼서 클릭 삼킴 현상을 완벽히 해결했습니다.

### 🚨 문제 2: 총알이 날아가지 않고 내 몸에서 (혹은 총구에서) 바로 터져버리는 현상
* **원인:** 캐릭터의 `AttemptShoot()` 함수 내부에서 연사 속도를 제어하는 **쿨타임(FireRate) 검사 코드가 주석 처리**되어 있었습니다. 이 때문에 마우스를 누를 때 1초에 60발의 총알이 거의 같은 위치에 동시에 스폰되었고, 총알끼리 물리적으로 충돌(Block)하여 스폰되자마자 파괴되었습니다.
* **해결 내용:** 쿨타임 로직(`if (CurrentTime - LastFireTime >= FireRate)`)을 다시 활성화하여 설정된 간격(0.1초)마다 한 발씩 정직하게 스폰되도록 수정했습니다.

### 🚨 문제 3: IMC_Default.uasset 관련 에러 폭탄 (A mapping cannot have an empty input action!)
* **원인:** 블루프린트 에디터에서 단축키 매핑(`IMC_Default`)을 설정할 때, 추가(`+`) 버튼만 누르고 액션을 지정하지 않거나, 모디파이어(Modifier)를 지정하지 않은 **빈칸(None)** 상태로 저장을 해서 엔진이 유효성 검사 에러를 뿜어낸 것입니다.
* **해결 내용:** `IMC_Default` 파일을 열어 `Mappings` 및 `Modifiers` 배열에서 비어있는(`None`) 껍데기 항목들을 휴지통 버튼으로 전부 깔끔하게 삭제하여 엔진의 에러 잔소리를 없앴습니다.

## 3. 총쏘기 구현 필수 지식 및 요약

### 📝 필수 지식 요약
1. **입력(Input)의 두 가지 철학**
   * **C++ 하드코딩 방식:** 에디터 설정 없이 코드로만 매핑을 강제하여 실수를 원천 차단 (1인 개발/학습에 유리)
   * **데이터 주도(Data-Driven) 방식:** C++에는 빈칸(`UPROPERTY`)만 뚫어두고, 에디터(블루프린트)에서 에셋(IA, IMC)을 드래그 앤 드롭으로 할당하는 방식 (게임 회사의 협업 정석)
2. **멀티플레이어 로직 분리 (Client vs Server)**
   * **클라이언트:** "나 총 쏠래!" 하고 마우스 클릭 감지 및 쿨타임 검사까지만 진행 (`AttemptShoot`)
   * **서버:** 요청을 받아 진짜로 총알을 스폰하고, 부딪혔을 때 파괴하는 물리 연산을 담당 (`ServerShoot_Implementation`, `HasAuthority()`)
3. **충돌 처리 (Collision Profile)**
   * 캐릭터 안에서 총알이 스폰될 때 캐릭터와 물리적으로 충돌해 튕겨 나가는 것을 막기 위해, 총알의 충돌체(Sphere Component)를 캐릭터 폰(Pawn)에 대해 **무시(Ignore) 혹은 겹침(Overlap)** 처리해야 합니다.

### 📌 한 줄 요약
> **"입력은 `GameAndUI` 모드로 확실히 받고, 쿨타임 검사로 무한 스폰을 막으며, 총알 스폰 및 파괴는 오직 `서버`에서만 처리하여 멀티 동기화를 달성한다!"**

## 4. 향후 구현 목표: 개별 슬롯 자동 충전 장전 시스템 (브롤스타즈 방식)

일반적인 슈팅 게임의 탄창식 장전(Magazine Reload)과 대비되는, 새롭게 도입할 장전 시스템의 핵심 설계 개념입니다.

### 🎯 핵심 용어
* **개별 슬롯 자동 충전 (Individual Slot Auto-Recharge)**
* **낱발 장전 / 단발 장전 (Single-Round Reload)**
* **독립 쿨타임 충전 (Independent Cooldown Recharge)**

### 📝 특징별 상세 설명

**1. 발사 즉시 자동 장전 시작 ("총알을 한 발 쏘면 오토 장전이 됨")**
* **시스템 설명:** 탄약을 소모하는 즉시 시스템이 이를 인식하여 자동으로 재장전 쿨타임(Cooldown)을 시작하는 '오토 리로드(Auto-Reload)' 방식입니다. 별도의 장전 버튼 입력이 필요하지 않습니다.

**2. 잔탄 유무와 무관한 독립적 충전 ("3발을 꼭 다 쏘지 않고 한 발만 쏴도 장전이 됨")**
* **시스템 설명:** 탄창을 완전히 비워야만 장전이 시작되는 것이 아닙니다. 탄창 전체가 하나의 묶음으로 처리되는 것이 아니라, **각각의 탄환 슬롯이 개별적인 쿨타임을 가지고 독립적으로 충전**됩니다.

**3. 장전과 발사의 비동기화 (행동 제어권 보장) ("장전 도중에 남은 총알도 쏠 수 있음")**
* **시스템 설명:** 장전 모션으로 인해 다른 행동이 제한되는 일반적인 방식과 다릅니다. **장전(충전) 프로세스와 발사 프로세스가 별개로 작동**하기 때문에, 빈 슬롯이 충전되는 도중에도 잔탄이 남아있다면 언제든지 즉시 발사할 수 있습니다.

### 💡 한 줄 요약
> **"탄창 전체를 교체하는 것이 아니라, 각 탄환 슬롯이 독립적인 쿨타임을 가지고 한 발씩 개별적으로 자동 충전되는 시스템"**
