# 탑다운 슈팅 게임 구현 가이드 (언리얼 엔진)

현재 프로젝트(`OpYB`)의 탑다운 슈팅 구현과 관련된 세 가지 핵심 원리와 디버깅 포인트를 정리한 문서입니다.

## 1. 언리얼 엔진의 탑다운 좌표계 이해
언리얼 엔진은 **왼손 좌표계 (Left-Handed Z-Up)**를 사용합니다.
* **X축 (빨간색)**: 캐릭터의 앞 (Forward) / 전진 방향
* **Y축 (초록색)**: 캐릭터의 오른쪽 (Right)
* **Z축 (파란색)**: 하늘 방향 (Up)

탑다운(Top-Down) 시점은 하늘(Z축 윗부분)에서 땅(XY 평면)을 내려다보는 구조입니다.
따라서 이동 로직(`MoveForward`, `MoveRight`)은 X축(`FVector(1, 0, 0)`)과 Y축(`FVector(0, 1, 0)`)으로 힘을 가해 바닥(XY 평면)을 평행하게 걷도록 구현됩니다.

---

## 2. 총알이 캐릭터 중심이 아닌 앞쪽에서 떨어져 스폰되는 이유
현재 `OpYBCharacter.cpp` 파일에는 다음과 같이 총알을 스폰하는 로직이 작성되어 있습니다.
```cpp
// 캐릭터 캡슐과 겹치지 않도록 스폰 거리를 150으로 벌림
FVector SpawnLocation = GetActorLocation() + GetActorForwardVector() * 150.0f + FVector(0, 0, 20.0f);
```

### 왜 굳이 밀어내서 스폰해야 할까요?
이것은 슈팅 게임 개발에서 반드시 처리해야 하는 **콜리전(충돌) 회피 기법**입니다. 
총알을 캐릭터의 정중앙(`GetActorLocation()`)에서 바로 스폰해 버리면, 총알이 생성되자마자 자기 자신(캐릭터의 캡슐 콜라이더)과 충돌하게 됩니다. 이로 인해 총알이 생성 즉시 자폭해서 사라지거나, 물리 엔진의 반발력 때문에 캐릭터가 뒤로 밀려나는 치명적인 버그가 발생합니다. 
이를 방지하기 위해 **총구 끝부분 쯤 되는 안전한 거리(앞쪽으로 150 유닛)**에서 스폰하도록 위치(Offset)를 더해준 것입니다.

## 3. 총알이 마우스 경로가 아닌 "캐릭터가 바라보는 방향"으로만 나가는 현상
현재 총알 발사 각도(회전값)를 결정하는 코드는 다음과 같습니다.
```cpp
FRotator SpawnRotation = GetActorRotation();
```
즉, **캐릭터의 현재 몸통 회전 방향**을 총알의 발사 방향으로 1:1로 그대로 가져다 쓰고 있습니다.

### 해결 및 수정 방향 (마우스 위치로 직접 발사)
만약 캐릭터의 회전 딜레이나 몸통 방향과 상관없이, **무조건 마우스 커서가 있는 3D 위치를 향해 정확히 날아가게** 하고 싶다면 회전값을 구하는 방식을 바꾸어야 합니다.
1. 발사 순간, `PlayerController`를 통해 마우스 커서의 3D 월드 위치(`TargetLocation`)를 구합니다.
2. `TargetLocation - 캐릭터 위치` 공식을 통해 방향 벡터를 얻습니다.
3. 이 방향 벡터의 회전값을 구해서 총알의 `SpawnRotation`으로 사용하도록 `ServerShoot()` 매개변수를 덮어씌워야 합니다.

### ※ 추가 체크: 허공을 클릭하면 총이 아예 안 나가는 경우
만약 "마우스를 캐릭터 위에 올려두고 클릭해야만 총이 쏴지고, 맨땅을 클릭하면 발사가 안 된다"면 이는 발사 방향 문제가 아니라 **입력 소비(Consume Input)** 문제입니다. 탑다운 템플릿의 기본 속성인 '클릭해서 이동(Click to Move)' 이벤트가 바탕화면 클릭 입력을 가로채고 있을 확률이 높으므로, 기본 마우스 이동 매핑을 프로젝트 셋팅이나 블루프린트에서 제거해주어야 합니다.
