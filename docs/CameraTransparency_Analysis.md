# 카메라 장애물 투명화(Camera Obstacle Transparency) 분석 보고서

본 문서는 `Operation: Breakout` 프로젝트에서 플레이어의 시야 확보를 위해 구현된 **카메라 장애물 투명화** 로직의 구조와 동작 원리를 분석한 자료입니다.

## 1. 개요
* **목적**: 탑다운 뷰 특성상, 캐릭터와 카메라 사이에 벽이나 큰 구조물이 위치할 경우 캐릭터가 가려져 조작 및 전투에 불편을 주는 문제를 해결합니다.
* **해결 방법**: `LineTrace`를 쏘아 카메라 시야를 가리는 장애물을 감지하고, 해당 장애물의 머티리얼(Material)을 다이내믹하게 변경하여 반투명(Opacity 조절) 상태로 만듭니다.

---

## 2. 작동 원리 (로직 흐름)

### ① LineTrace(라인 트레이스)로 오브젝트 감지
매 프레임(Tick) 혹은 일정한 주기로 **카메라의 위치(Start)**에서 **캐릭터의 위치(End)**로 보이지 않는 선(Line Trace)을 발사합니다.
* **채널 설정**: 이 선은 바닥이나 캐릭터 자체는 무시하고, '장애물(Wall, Prop 등)' 콜리전 채널만 감지하도록 설정됩니다.
* 선에 부딪힌 오브젝트(`HitResult`)가 존재한다면, 그것이 현재 시야를 가리고 있는 장애물임을 의미합니다.

### ② Dynamic Material Instance 생성 및 적용
언리얼 엔진에서 머티리얼의 투명도를 실행 중에 변경하려면 **다이내믹 머티리얼 인스턴스(UMaterialInstanceDynamic)**가 필요합니다.
* 감지된 장애물의 기존 머티리얼을 다이내믹 인스턴스로 교체하거나, 미리 설정된 다이내믹 인스턴스의 파라미터(예: `Opacity`) 값을 낮춥니다 (예: 1.0 ➔ 0.3).

### ③ 상태 복구 (Restore)
장애물이 더 이상 시야를 가리지 않게 되면(캐릭터가 이동하여 LineTrace에 아무것도 걸리지 않게 되면), 다시 원래의 투명도(Opacity = 1.0)로 서서히 혹은 즉시 복구시킵니다.

---

## 3. 요약 및 장점
* **플레이어 경험 향상**: 시야가 가려지는 답답함을 완벽하게 해소하여 쾌적한 전투 환경을 제공합니다.
* **정밀한 타겟팅**: 투명화된 장애물 너머로도 적을 확인하고 스킬이나 사격을 정확하게 이어갈 수 있어 배틀로얄 장르에 필수적인 기능입니다.
