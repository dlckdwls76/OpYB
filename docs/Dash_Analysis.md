# 스페이스바 점멸(Dash / Blink) 기능 분석 보고서

본 문서는 `Operation: Breakout` 프로젝트에서 구현된 **캐릭터 점멸(회피기)** 로직의 구조와 동작 원리를 분석한 자료입니다.

## 1. 개요
* **목적**: 좁혀오는 자기장과 적의 투사체를 순간적으로 회피하거나, 유리한 포지션을 선점하기 위한 기동성 기술.
* **조작**: `스페이스바 (Spacebar)` 입력.
* **특징**: 무한정 사용할 수 없도록 쿨타임(Cooldown)이 존재하여 전략적인 판단을 요구합니다.

---

## 2. 작동 원리 (로직 흐름)

### ① 입력 감지 및 상태 확인
스페이스바를 누르면 언리얼 엔진의 Input 액션이 발생합니다.
* `bCanDash` 등의 boolean 변수를 통해 현재 쿨타임 중이 아닌지(점멸 사용 가능 상태인지) 먼저 검사합니다.

### ② 목표 지점 계산
현재 캐릭터가 이동 중인 방향(Input Vector) 혹은 바라보고 있는 방향(Forward Vector)을 기준으로 일정 거리(예: 500 유닛) 앞의 목표 지점을 계산합니다.

### ③ 순간 이동 (Teleport / Launch)
목표 지점을 향해 캐릭터를 이동시킵니다.
* **방식 A (LaunchCharacter)**: 물리적인 힘을 가해 빠르게 미끄러지듯 돌진(Dash)시킵니다. (부드러운 연출)
* **방식 B (SetActorLocation)**: 특정 좌표로 즉시 순간 이동(Blink)시킵니다.
현재 프로토타입은 빠르고 직관적인 회피를 위해 즉시 이동/빠른 돌진 로직을 기반으로 동작합니다.

### ④ 쿨타임(Timer) 적용
점멸 직후 `bCanDash` 변수를 `false`로 변경하여 연속 사용을 막습니다.
* **TimerManager**를 사용하여 지정된 쿨타임(예: 3초 ~ 5초)이 지난 후 `ResetDash()` 함수를 호출, 다시 `bCanDash`를 `true`로 돌려놓습니다.

---

## 3. 요약 및 장점
* **전투 템포 증가**: 단순히 걷고 총을 쏘는 것을 넘어, 순간적인 회피기를 통해 게임의 텐션과 생존력을 끌어올립니다.
* **네트워크 동기화 주의**: 점멸 위치는 서버에서 승인(RPC)을 받아야 클라이언트 간 위치 버그(고무줄 현상) 없이 부드럽게 모든 유저에게 동기화됩니다.
